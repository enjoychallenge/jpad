Code Modules

Google Closure Compiler and Google Closure Library enable to split your application into more code modules (standalone JS files). Andvantage is you can download code of some complex component when you really need it. However there are some limitations.

If you want to use these code modules, you need to write your app in quite different way. Without modules you need exactly one input point for compiler. With code modules you need one input point per module. And there is more. If you compile your application into multiple code modules, you cannot compile it in RAW mode and source maps are broken. This makes debugging very painful.

This example shows how you are able to combine both approaches (with or without modules) with jpad. Changing jpad.ENABLE_MODULES variable enables/disables code modules on without any other changes in your code. If you want to debug, just disable code modules and you can compile it in RAW mode or use source maps.
